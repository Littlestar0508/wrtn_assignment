## STEP1. 기본 환경 설정

- Zustand 사용
- TypeScript 사용
- TailwindCSS 사용
- alias 루트 설정 (`@/*`)
- title 태그 template 설정
- TailwindCSS 디자인 토큰 설정

- Zustand 선택 이유 : 전역 상태를 직관적으로 관리하고 새로고침에도 사용자 설정을 유지하기 위해 사용했습니다.

## STEP2. 지역 선택

- select option 태그를 활용한 드랍다운 구현
- 서울특별시, 경기도를 제외한 지역 선택 후 다음 버튼 클릭 시 alert 표시 및 이동 제한
- 서울특별시, 경기도 선택 시 다음 화면 이동 여부 확인 후 이동

---

## STEP3. 기본 요금제 설정

- 라디오 컴포넌트를 통한 단일 옵션 선택 구조 구현
- Zustand 기반 상태 관리
- persist 패턴을 활용해 새로고침 시에도 설정 유지
- 모든 설정 완료 전 가격 렌더링 제한
- 스마트 미터기 구매 여부 체크박스 제공
- 사용량을 알고 있는 사용자를 위한 직접 입력 페이지 이동 링크 제공
- 설정 미완료 상태에서 다음 버튼 클릭 시 경고 표시
- 고정 요금제: 1년, 2년, 3년 요금 렌더링
- 가변 요금제: 여름, 겨울, 평상시 월별 요금 렌더링
- 설정 완료 후 다음 버튼 클릭 시 결제 페이지 이동
- 계산 로직: `BasicCalculate.ts`

## STEP4. 소비량 기반 설정

- 소비량 입력 input 제공
- 소비량 인지 여부 선택 라디오 컴포넌트 제공
- 스마트 미터기 구매 여부 체크 유지
- 모든 설정 완료 전 가격 렌더링 제한
- 다음 버튼 클릭 시 조건별 경고 처리
  - 소비량 모름 선택 시 STEP3 이동 안내
  - 소비량 0 입력 시 입력 요청
  - 기타 설정 미완료 시 설정 요청
- 모든 설정 완료 후 결제 페이지 이동
- 계산 로직: `SmartCalculate.ts`

## STEP5. 요금제 비교 및 결제

- 고정 요금제, 가변 요금제 선택 UI 제공
- 연도별 요금 계산 구조 적용
- 가변 요금제는 평균 단가 120원 기준 계산
- 선택 시 연도별 계산 결과 모달 팝업 표시
- 모달 UX
  - ESC 키로 닫기
  - dimmed 영역 클릭 시 닫기
  - 닫기 버튼 제공
- Zustand 상태로 모달 열림/닫힘 관리
- `totalElectric`, `totalPrice` 계산 로직 분리
- 모달 결제 버튼 클릭 시 결제 완료 페이지 이동
- 결제 완료 페이지에서 선택 정보 기반 결과 렌더링
